INFO

This application is build to receive Webhook messages from Mist and forward them to Slack/Teams. To do so, it requires some configuration.
It will automatically configuring the Mist Webhooks in your organization based on the topics you enable.

Webhook Topics

This section allows you to configure which webhook topics you want to forward to Slack/Teams and how the system is processing the webhooks.
To add a new topic, click the "Add Webhook Topic" button and select the topic you want. From there, you will be able to assign a "channel" to each event type:
  • "disabled": the event will be discarded by the server, and will not be forwarded to Slack/Teams
  • all the others: the event will be processed by the server, and will be forwarded to Slack/Teams
Note about "audit" topic
The "audit" topic allows you to list a number of "approved" administrators and API Tokens. This will allow you to differentiate actions done by "approved" administrators/API Tokens and actions done by others. For example, you can "disable" the "approved_admins" and send the "other_admins" notifications in the "warning" channel to get warned when non approved admins are doing changes in the organization.
To add new "approved" administrators/API Tokens, just add their email address or the API Token key in the "Approved Admins" field.

Notification

This section allows you to configure how the system is forwarding the messages to your Slack/Teams account. To do so, it is using "Incoming Webhooks" for both of them, which means the system send a HTTPS POST message to a specific URL, each URL being tied to a Slack/Teams channel.
To configure Slack/Teams, click the "Add Notification" button and configure the incoming webhook URLs:
  • it is required to configure one URL for each channel you want to received ("debug", "info", "warning", "critical"). The "default" channel is mainly used to send events that are not yet listed by this application.
  • all of them are optional, but if you are not configuring them, you will not receive the messages assigned to the corresponding channel
